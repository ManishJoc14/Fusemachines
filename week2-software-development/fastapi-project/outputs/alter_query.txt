seed.sql had limitations:

customerNumber was manually assigned (not auto generated)
foreign key constraints blocked deletes 

To fix this, we applied:

=> Auto-generate customer ID:
ALTER TABLE customers
ALTER COLUMN "customerNumber"
ADD GENERATED BY DEFAULT AS IDENTITY;



=> Enable cascade delete for payments:
ALTER TABLE payments
DROP CONSTRAINT "payments_customerNumber_fkey";

ALTER TABLE payments
ADD CONSTRAINT payments_customerNumber_fkey
FOREIGN KEY ("customerNumber")
REFERENCES customers ("customerNumber")
ON DELETE CASCADE;


=> Enable cascade delete for orders:
ALTER TABLE orders
DROP CONSTRAINT "orders_customerNumber_fkey";

ALTER TABLE orders
ADD CONSTRAINT orders_customerNumber_fkey
FOREIGN KEY ("customerNumber")
REFERENCES customers ("customerNumber")
ON DELETE CASCADE;



=> Result:
DB now auto-generates IDs
Deleting a customer automatically removes related orders and payments
